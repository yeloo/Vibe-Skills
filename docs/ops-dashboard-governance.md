# Ops Dashboard Governance

## Purpose

Wave95 packages a lightweight ops dashboard so operators can read Wave83-100 evidence without manually stitching together verify outputs, board data, and release notes.
The dashboard reads governed evidence and summarizes it; it does not invent stages or become a second control plane.

## Governance Scope

- The dashboard must expose `freshness`, `promotion`, `replay`, `rollback`, and `release` panels.
- Every panel must point back to governed sources such as promotion board, release ledger, and verify artifacts.
- The dashboard is regenerated by the gate; it should not be hand-maintained as a free-form report.

## Required Invariants

- `reads_governed_evidence_only = true` remains fixed.
- `panel_ids = freshness,promotion,replay,rollback,release` remain fixed.
- The dashboard cannot claim release readiness if upstream evidence is missing.
- Operator-readable markdown and machine-readable JSON must be emitted together.

## Non-Goals

- No second board is created inside the dashboard.
- No panel may mutate promotion or routing state.

Validation is performed by `scripts/verify/vibe-ops-dashboard-gate.ps1`.
