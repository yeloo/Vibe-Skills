## Highlights

- 完成 Wave19-23 的 memory / prompt absorption 收口：`mem0` 被固定为可选 preference backend，`Letta` 被固定为 policy contract 来源，prompt intelligence 只增强 pattern cards / risk checklist，不形成第二 router。
- 完成 Wave24-25 的执行平面吸收：BrowserOps 以 provider governance 方式吸收 `browser-use`，DesktopOps 以 shadow contract 方式吸收 Agent-S，二者都保留 `advice-first / shadow-first / rollback-first` 约束。
- 完成 Wave26-28 的治理闭环：新增 cross-plane conflict policy、promotion board、pilot fixture 体系，并把四个平面的试点评估入口固定到 `pilot-*.json`。
- 完成 Wave29 的 mirror / packaging 收口：canonical 与 `bundled/skills/vibe` 镜像重新同步，`version-packaging` 与 `config-parity` 门禁通过，同时移除已退役的 `*-shadow.json` pilot 旧命名，降低冗余和歧义。
- 完成 Wave30 的最小 promotion 与 release cut：`memory-runtime-v2`、`prompt-intelligence`、`browserops-provider`、`desktopops-shadow` 已由 `shadow` 显式写入 `soft`，并产出新的 changelog / ledger / release note。

## Migration Notes

- 本次发布不改变 VCO 的三条核心不变量：**不引入第二 orchestrator、不引入第二 canonical truth-source、不引入第二 default execution owner**。
- `browser-use` 仍然只是 BrowserOps provider candidate；`Agent-S` 仍然只是 DesktopOps shadow contract 来源；两者都不能接管 pack selection、task decomposition、team orchestration 或 promotion decision。
- 新的 pilot 入口只认 `scripts/verify/fixtures/pilot-memory.json`、`scripts/verify/fixtures/pilot-prompt.json`、`scripts/verify/fixtures/pilot-browserops.json`、`scripts/verify/fixtures/pilot-desktopops.json`；旧的 `*-shadow.json` 不再是可执行试点资产。
- 如果需要回滚 promotion board，可把四个 plane 的 `current_stage` 从 `soft` 写回 `shadow`，并按各自 `rollback.command` 执行最小回退。
