## Scholarly Publishing / Figures / Reporting（详细说明）

本节用于解释：为什么要新增学术写作/作图/报告相关的 pack/skill、它们解决了什么问题、如何稳定激发，以及你会得到什么交付物。

### 1) 解决的问题（Why）

过去在“写报告 / 论文作图 / 投稿返修”等场景里，路由器容易被这些低信息词带偏：
- 文件类型：`pdf` / `docx` / `word`
- 泛化动作：`写报告` / `做图` / `导出`

结果是：
- Prompt 被误判为“纯文档处理”，路由到 `docs-media`（只擅长 PDF/DOCX 提取/格式处理）。
- 缺少顶刊级规范（figure 标准、投稿 checklist、rebuttal 资产、可复现构建与打包）。

本次更新的目标是：把“学术写作/作图/报告”从“文档格式处理”中分离出来，变成**可交付的出版资产工程**（manuscript / figures / submission / revision / report / slides）。

### 2) 新增 Packs（路由层，What routes where）

Pack 定义与触发词见：`config/pack-manifest.json`（bundled 镜像：`bundled/skills/vibe/config/pack-manifest.json`）。

| Pack | 典型触发（例） | 主要解决什么 | 默认会拉起的技能（候选集的一部分） |
|---|---|---|---|
| `scholarly-publishing-workflow` | `投稿`/`submission`、`返修`/`revision`、`rebuttal`、`camera-ready`、`proof` | 论文从草稿 → 投稿 → 返修 → 相机就绪的全流程资产化 | `scholarly-publishing`、`submission-checklist`、`latex-submission-pipeline`、`manuscript-as-code` |
| `science-figures-visualization` | `顶刊作图`、`publication-quality`、`panel`/`多子图`、`TIFF 600dpi`、`colorblind` | 数据图/多面板图的顶刊级规范与导出（矢量/栅格） | `scientific-visualization`、`scientific-schematics`、`scholarly-publishing` |
| `science-reporting` | `科研报告`、`技术报告`、`实验报告`、`HTML 报告`、`PDF 报告`、`Quarto` | 报告结构化（Executive Summary/Methods/Results/Appendix）+ 图表规范 | `scientific-reporting`、`scientific-writing`、`scientific-visualization`、`slides-as-code` |

### 3) 新增 Skills（执行层，你会得到什么交付物）

这些技能不只是“写一段文字/画一张图”，而是面向可提交与可复用的**资产包**。

- `bundled/skills/scholarly-publishing/SKILL.md`
  - 输出契约：`manuscript/`、`figures/`、`supplement/`、`submission/`、`revision/`、`build/` 的出版资产目录与规范。
  - 适用：投稿/返修/相机就绪/顶刊作图/LaTeX 工程化/组会汇报串联。

- `bundled/skills/submission-checklist/SKILL.md`
  - Stage-based checklist：pre-submission / submission / revision/rebuttal / camera-ready。
  - 提供模板：`bundled/skills/submission-checklist/templates/*`（cover letter、rebuttal matrix、camera-ready checklist）。

- `bundled/skills/manuscript-as-code/SKILL.md`
  - 把论文当软件：版本控制、生成物与源分离、可复现构建、固定仓库结构（避免“final_v7”）。

- `bundled/skills/latex-submission-pipeline/SKILL.md`
  - LaTeX 构建与投稿打包：`latexmk`、BibTeX/Biber、`chktex`、`latexindent`、CI 编译、submission zip。
  - 模板入口：`bundled/skills/latex-submission-pipeline/templates/*`。

- `bundled/skills/slides-as-code/SKILL.md`
  - Slidev/Marp/Reveal/Quarto 的 slides 工程化：可复现导出、图表复用（不截图）、一页一结论。
  - 适用：组会/答辩/汇报/论文传播。

- `bundled/skills/scientific-reporting/SKILL.md`
  - 报告结构与图表规范一体化：Markdown source-of-truth → 可选 HTML/PDF；带 Executive Summary 与可复现 Appendix。
  - 报告骨架模板：`bundled/skills/scientific-reporting/templates/report-skeleton.md`。

案例库（高信号 GitHub 结构参考）：`bundled/skills/scholarly-publishing/references/case-library.md`。

### 4) 如何稳定激发（Activation / 调用建议）

如果你希望路由器更稳定命中学术域 pack，请在 prompt 中加入“域信号词”，避免只写文件格式。

建议直接复制以下任意一句作为开头：

- 投稿/返修：
  - “我要 **投稿**，请按 **submission checklist** 帮我整理投稿包（cover letter + 文件清单 + 打包规范），并给出需要补齐的信息。”
  - “我要写 **rebuttal / 回复审稿意见**，请输出逐条回应矩阵（comment → action → response → location）。”

- 顶刊作图：
  - “我要做 **publication-quality 多子图(panel) figure**，要求统一字体线宽、色盲友好，并导出 **PDF + TIFF 600dpi**。”

- 科研/技术报告：
  - “我要写 **科研/技术报告**（面向：老板/评审），输出 **Executive Summary + Methods + Results + Appendix**，最终导出 **HTML + PDF**。”

- Slides：
  - “我要把论文做成 **Slidev/Marp slides**，要求图表不截图，直接复用 figures 的最终导出，并能导出 PDF。”

如果你要强制指定 skill，也可以在 prompt 中显式点名（按 Rule 3：用户指令优先）：
- `$scholarly-publishing` / `$submission-checklist` / `$scientific-reporting` / `$slides-as-code`
